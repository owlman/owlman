## 我在 GitHub 上做什么？

由于我是一名技术写作者与计算机系统学习者，专注长期积累与可复用的技术判断，因而在这里主要致力于整理计算机基础、工具链取舍、写作方法与实验记录，具体包括如下范畴：

1. **计算机科学领域的长期笔记**
   - 算法 / 操作系统 / 编译原理等基础路线与笔记
   - 语言学习体会：Python、JavaScript、C++ 等

2. **开发工具与工程实践**
   - 前端与后端框架：Node.js、Vue.js、uni-app 的实践总结
   - 编辑器与 IDE：NVim、VS Code、OpenCode 使用经验
   - 运维与部署：Git、Docker、Kubernetes 等工具的实战记录

3. **技术写作与可复现实验**
   - 使用 Obsidian / Markdown / LaTeX 作为写作工具
   - 记录完整实验过程（环境、命令、结果），便于复现
   - 把学习笔记打磨成有判断力的文章

我的目标是确保自己在 GitHub 上分享的内容具有长期价值，能够帮助读者建立系统性的知识体系，并具备可复现性。为此，我也会尽可能地避免在这里做以下事情：

- 产出标题党或情绪化的内容
- 把 AI 当作替代思考的工具
- 追逐短期热点或流量红利
- 假装自己无所不知

### 适合谁关注我

- 想知道「为什么」而不仅是「怎么做」的读者
- 需要完整思路、权衡过程与失败记录的人
- 注重长期积累与系统学习而非速成的学习者
